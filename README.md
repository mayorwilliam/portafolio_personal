# Developer Portfolio v1

A high-performance, tech-focused portfolio built with Next.js 16, Tailwind CSS, and Notion as a CMS.

## 🚀 Quick Start

1. **Install Dependencies**:
   ```bash
   npm install
   ```

2. **Configure Environment**:
   ```bash
   cp .env.example .env
   # Add your Notion Keys in .env (See below)
   ```

3. **Run Locally**:
   ```bash
   npm run dev
   ```

## 📝 Notion CMS Setup (Required for Real Data)

To make the portfolio dynamic, you need to create two databases in Notion.

### 1. Get your API Key
1. Go to [https://www.notion.so/my-integrations](https://www.notion.so/my-integrations).
2. Click **New integration**.
3. Name it "Portfolio CMS".
4. Copy the **Internal Integration Secret**.
5. Paste it as `NOTION_TOKEN` in your `.env` file.

### 2. Create "Projects" Database
Create a new Table View database and add these exact columns (properties):

| Property Name | Type | Description |
| :--- | :--- | :--- |
| **Title** | Title | Name of the project |
| **Slug** | Text | URL-friendly ID (e.g. `my-awesome-app`) |
| **Description** | Text | Short summary |
| **TechStack** | Multi-select | Tags like `React`, `Node`, `AWS` |
| **DemoURL** | URL | Link to live site |
| **RepoURL** | URL | Link to GitHub |
| **Image** | Files & Media | Cover image for the card |
| **Video** | URL | Link to mp4 or video demo |
| **Featured** | Checkbox | Check to show on Home page |

### 3. Create "Experience" Database
Create another Table View database with these columns:

| Property Name | Type | Description |
| :--- | :--- | :--- |
| **Role** | Title | Job Title (e.g. `Senior Engineer`) |
| **Company** | Text | Company Name |
| **Period** | Text | e.g. `2023 - Present` |
| **Description** | Text | Bullet points separated by new lines |

### 4. Connect Integration & Get IDs
1. Open your new `Projects` database page.
2. Click `...` (top right) -> `Connections` -> `Connect to` -> Select your **Portfolio CMS** integration.
3. Repeat for `Experience` database.
4. **Get Database ID**:
   - Copy the URL of the database.
   - It looks like: `https://notion.so/myworkspace/a8aec43384f447ed84390e8e42c2e089?v=...`
   - The ID is the part **before** the `?`. In this example: `a8aec43384f447ed84390e8e42c2e089`.
   - Paste this as `NOTION_PROJECTS_DATABASE_ID` in `.env`.
   - Do the same for `NOTION_EXPERIENCE_DATABASE_ID`.

## 🛠 Tech Stack

- **Framework**: Next.js 16 (App Router)
- **Styling**: Tailwind CSS v4
- **CMS**: Notion API

## 🐳 Deployment (Dokploy)

1. **Push to GitHub**.
2. **Setup in Dokploy**:
   - Select "Docker Compose" deployment.
   - Add your `NOTION_TOKEN` and Database IDs in the Environment Variables section.
